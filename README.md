# 🍏 Hackintosh | DELL OptiPlex 5070 MFF For MacOS 26 Tahoe 版本的 EFI 存档

> **写在前面**：全网搜了一圈，截止目前应该还是没有发现这个机型（限我这个配置，其他系列有，但是跟我这个差别挺大，没法用）运行于 MacOS Tahoe 26 版本的 EFI 存档，而且一路升级过来，从最初的 26.0 一直升级至目前最新的 26.4 版本，都能完美运行。之前我存档的该机型的两个版本的 EFI（12 和 14）均有 Wi-Fi 问题，因为我是直接下载别人的来用，也修改过一些配置，但是没有深层次修改。而这次是因为直接找不到对应能用的版本，至少网络上的几个版本（2025 年底）的那几个，声称是完美运行的，也有跟我这个机器配置接近的，但是都是有问题，没法使用，有的直接没法开机。最终静下心来研究了两周，跑了国外各大论坛，包括 Reddit 和 insanelymac，学习了无数大佬的修改方案，最终基于黑果小兵的自动化生成工具 easyEFI，自己生成了一套 EFI 文件，但是也无法直接使用，看了很多的帖子，最终在 insanelymac 一个帖子上受到启发，修改出来了完美的 EFI 文件（至少在我这台机器上算是完美了）。但是由于硬件过于落后，运行 Tahoe 已经比较缓慢了，不过我也能等，作为主力机就这样一直用了三个月，现在看还是没人分享，我就发布出来给大家分享使用。

## 📝 项目与运行状态说明

* **运行状态**：核心功能正常，长期使用稳定性极佳，没有出现过大的问题。
* *💡 必须安装：苹果从 macOS Sonoma (14.0) 开始，在底层彻底删除了对大部分博通网卡的免驱支持。因此在 14.x 之后以及此次的 26.x 系列系统中出现问题是正常现象，必须需要安装 OCLP-Mod (目前最新的是 3.1.9 版本，如果你升级到了 26.4 版本必须装到最新版本) 强制打上 Root Patcher 补丁才能恢复。*
* *💡 安装步骤：设置 ➡️ 补丁，勾选“允许安装旧版HDA环境扩展补丁”，勾选“允许安装旧版USB环境扩展补丁”，退出到主界面后，点击“安装驱动补丁”，等待安装完成并重启，你的蓝牙和 Wi-Fi 就回来了。同时注意，每次升级系统版本，比如 26.3升级到26.4，必须重新执行上述补丁操作，否则蓝牙和 Wi-Fi 就失效了。*
